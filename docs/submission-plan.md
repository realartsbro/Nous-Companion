# Nous Companion — Hackathon Submission Plan

## Strategy

**Submission requirements:**
- Tweet a demo video tagging @NousResearch
- Short writeup in the tweet
- Link in `?creative-hackathon-submissions` channel
- Kimi track: prove Kimi model usage in the video

**Core pitch:** Nous Companion is an _always-on desktop companion_ for Hermes Agent users — animated portraits, reactive quips, lip-synced TTS. It's the missing "personality layer" for the Hermes ecosystem.

**Tone:** Warm, playful, slightly mysterious. The companion IS the product — let her charm speak for itself.

## Demo Video Script (~90s)

### [0:00-0:08] — Opening Shot
**Visual:** Clean desktop. A small window appears — dark teal with "NOUS COMPANION" in Collapse Bold. The brand spotlight sweeps across the text. Splash fades to the animated character portrait (eyes blink, idle expression shifts).

**Audio:** Soft ambient music fade in.

**Text overlay (bottom third):** `A desktop friend for Hermes Agent`

### [0:08-0:20] — Meet the Companion
**Visual:** The character portrait cycles through expressions — normal → smiling → serious. Scanlines, grain, and analog bleed visible. The status dot pulses cream.

**Action:** Switch characters using the ring selector (◀ ▶ arrows). Portrait changes with a burst flash transition.

**Audio:** Subtle TTS: "I see you're working. Want some company?" (whichever character is active)

### [0:20-0:35] — Reactive Quips
**Visual:** Split screen — Hermes terminal on the left, companion on the right. User types a Hermes command. The companion's expression shifts, godmode text streams at the bottom.

**Action:** Show 2-3 reaction examples:
- File written → "Clean save. Good instincts."
- Code search → "Looking for something?"
- Error → brief frown

**Audio:** TTS reactions play with visible lip sync.

### [0:35-0:50] — Settings & Customization
**Visual:** Open the settings overlay (click portrait). Quick tour:
1. QUICK tab → character ring selector, volume
2. CHARACTER tab → expression groups, idle rarity dots
3. DISPLAY tab → toggle Hermes/Classic mode, toggle effects (scanlines, burst, grain)
4. SYSTEM tab → TTS engine, model selection

**Shorthand:** Fast cuts, 2-3 seconds per tab.

### [0:50-1:00] — ✨ Kimi Track Proof
**Visual:** SYSTEM settings tab. Open the model dropdown. Scroll to and select a Kimi model (e.g., `kimi/k2.5-32k-thinking`).

**Action:** Send a prompt to Hermes. The companion reacts — generated by Kimi.

**Text overlay:** `Powered by Kimi ✦` (appears briefly)

**Audio:** TTS quip generated by Kimi model.

### [1:00-1:10] — Classic Mode Easter Egg

## Kimi Track Qualification

To qualify: the submission video must **prove Kimi model usage**.

**Method 1 (best):** Show the companion routing to a Kimi model in real-time. Open SYSTEM tab, select a Kimi model from the dropdown, send a Hermes command, show the companion reaction generated by Kimi.

**Method 2 (backup):** In the companion_server.py config or settings, hard-code a Kimi model as the current provider. Show the companion's quip appearing with "kimi" in the logs or model selection UI.

**Method 3 (simple):** Record the settings page where the model dropdown shows a Kimi model selected. The text "Powered by Kimi" overlay seals it.

**Recommended Kimi models to use:**
- `kimi/k2.5-32k-thinking` — thinking/reasoning model
- `kimi/k2.5-72b` — fast general purpose

We have these production pipelines available:
